# 지구온난화 시뮬레이터 제작 노트

중학교 3학년 과학 수업용 기후 정책 시뮬레이터의 설계 의도, 모델 구조, 구현 조건을 정리한 문서다.

---

## 기본 방향

- **단일 HTML 파일** 원칙: 외부 라이브러리 없음, 인터넷 없이 실행 가능
- En-ROADS 기후 정책 시뮬레이터에서 착안한 중학생용 축약 모델
- 정책 선택 → 배출량 → 농도 → 기온의 인과 흐름을 직접 체험하는 것이 목표
- 20분 수업 활동에 맞게 4턴(2030·2040·2060·2080)으로 압축
- 학생 개인정보 없음, 도시명만 입력

---

## 게임 구조

### 턴 구성
- 4개 정책 턴: 2030 → 2040 → 2060 → 2080
- 최종 결과: 2100년 기온·농도·발전도 표시
- 이벤트: 2040(전력 수요), 2060(숲 보호) 각 1회

### 목표 수치
- 기온 변화 **2.0°C 이하**
- 도시 발전도 **70점 이상**

두 목표 동시 달성은 의도적으로 어렵게 설계함 (최적 전략으로 약 1.99°C 달성 가능, 발전도 85점). 상품을 걸 수 있는 수준의 난이도.

## 정책 시스템

### 6개 정책

| 정책 | 배출 감소 계수 | 흡수 계수 | 예산/레벨 | 발전도/레벨 |
|------|--------------|----------|----------|-----------|
| 화석연료 사용 줄이기 | 0.12 | - | 7 | -2 |
| 재생에너지 확대 | 0.09 | - | 9 | +5 |
| 에너지 효율 기술 향상 | 0.08 | - | 7 | +6 |
| 산림 보호·복원 | - | 0.06 | 5 | +2 |
| 도시 에너지 소비 줄이기 | 0.08 | - | 4 | -4 |
| 탄소 제거 기술 투자 | - | 0.05 | 10 | +3 |

- 레벨 범위: 0~4
- 정책은 다음 턴으로 누적 유지됨 (레벨 낮추는 것도 가능)

### 장관 승인 시스템
- **레벨 3 이상** 정책은 해당 부처 장관 전원 승인 필요
- **경제부 장관**: 6개 정책 전부 관할 → 자연스러운 병목
- **장관별 턴당 승인 2회 한도**: 토론을 유도하는 핵심 제약
  - 경제부 2회 한도 = 실질적으로 턴당 고강도 정책 2개 제한

### 장관별 관할
| 장관 | 관할 정책 |
|------|----------|
| 환경부 | 화석연료, 산림, 에너지소비, 탄소제거 |
| 경제부 | 전체 6개 |
| 과학기술부 | 효율 기술, 탄소제거 |
| 에너지·산업부 | 화석연료, 재생에너지, 에너지소비 |

## 과학 모델

### 배출량 계산
```
directCut = 각 정책의 directEmissions 계수 × 레벨의 합
emissions = baseline[year] × max(0.18, 1 - directCut)
```
- 배출량 하한선 18%: 현실적 감축 한계 반영
- 이벤트 페널티(fossilBackup 선택 시): `+ baseline × 0.04` (2040년 이후 매년)

### 농도 계산
- 첫 턴(2030): `baseline[2030] - (배출 감소 기여 × 12 + 흡수 기여 × 10)`
- 이후 턴: `이전 농도 + baselineStep × (배출/기준배출) × max(0.42, 1 - sinkSupport)`
  - sinkSupport: 산림 레벨 × 0.06 + 탄소제거 레벨 × 0.05

### 기온 계산
```
concentrationGap = 현재농도 - baseline농도[year]
target = baseline온도[year] + concentrationGap × 0.008
temperature = 이전기온 + (target - 이전기온) × 0.7
```
- 0.7 지연 계수: 기온이 농도에 시간차를 두고 반응하는 열관성 표현

### 기준(무정책) 경로
| 연도 | 배출량(Gt) | 농도(ppm) | 기온(°C) |
|------|-----------|----------|---------|
| 2030 | 57 | 435 | 1.5 |
| 2040 | 60 | 470 | 1.8 |
| 2060 | 62 | 535 | 2.5 |
| 2080 | 61 | 595 | 3.1 |
| 2100 | 59 | 650 | 3.6 |

### 발전도 계산
- 시작값: 50점
- 정책 레벨 × 발전도 계수 누적 합산
- 이벤트 선택 결과 별도 가산

## 이벤트 시스템

이벤트 효과는 두 가지 경로로 반영됨:
- **emissionsModifier**: 시뮬레이션 배출량에 영구 가산
- **renewablesBoost / efficiencyBoost / forestsBoost**: 시뮬레이션 내부에서만 정책 레벨에 가산 (슬라이더 미반영)
- **development / budget**: `addEventScores()`로 별도 합산

슬라이더를 건드려도 이벤트 효과는 취소되지 않음 (시뮬레이션 레이어에서 적용).

### 2040 이벤트: 전력 수요 급증
| 선택 | 예산 | 발전도 | 배출 영향 |
|------|------|--------|---------|
| 화석연료 일부 유지 | 2 | +3 | +0.04 배출 계수 영구 적용 |
| 재생에너지 확대 | 9 | +4 | 재생에너지·효율 각 +1 (시뮬레이션) |

### 2060 이벤트: 숲 보호
| 선택 | 예산 | 발전도 | 배출 영향 |
|------|------|--------|---------|
| 개발 우선 | 4 | +7 | 산림 -1 (시뮬레이션) |
| 숲 보호 | 6 | +3 | 산림 +1 (시뮬레이션) |

## 도시 그래픽 설계

Canvas 2D로 직접 렌더링. 정책 상태에 따라 실시간 변화.

| 시각 요소 | 반응하는 정책 |
|----------|------------|
| 하늘 색 (열기) | 기온 변화 |
| 태양광 패널 | 재생에너지 + 효율 |
| 나무 밀도 | 산림 |
| 매연 구름 | 화석연료 감소 |
| 건물 창문 점등률 | 에너지소비 감축 |
| 가로등 밝기 | 에너지소비 감축 |
| 태양 크기·색 | 기온 변화 |
| 전기차 비율 | 재생에너지 + 효율 |

## 수업 설계 의도

- **토론 우선**: 장관 승인 2회 제한은 정답 찾기가 아닌 어떤 정책에 우선순위를 줄지 토론을 유도하기 위한 장치
- **역할 분담**: 4개 부처 역할이 학생들에게 배분되므로 승인 결정에 책임감 부여
- **상품 조건**: 기온 2.0°C 이하 + 발전도 70 이상 동시 달성 → 최적 전략을 알아도 예산 관리가 정교해야 하므로 변별력 있음
- **개인정보 없음**: 도시명만 저장, PNG 보고서로 종료
